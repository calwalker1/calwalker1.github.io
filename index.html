The better playback tool

Hopper plays back an archival recording of your show and keeps the video locked to a live ETC EOS lighting console over OSC. When the operator fires cue 40, the video jumps to the exact moment cue 40 ran in the recording.

Scrubbing through long videos to find cues is slow, disruptive, and pulls focus away from the work at hand. Hopper removes that friction—the moment a cue fires on the console, the picture is already there.

How it works

From import to session in four steps.

  1. 1

    Import your recording

    Open any video file of your show. Hopper supports multi-act productions—load an entire run as a sequence and it jumps between videos automatically.

  2. 2

    Draw a detection region

    Drag a box over the burnt-in cue number on screen. Hopper uses on-device OCR to read the cue display frame by frame and builds a precise map of every cue to its exact timestamp in the video.

  3. 3

    Connect your console

    Point your console’s OSC output at Hopper. UDP and TCP are both supported. No cue-level programming is required on the console—Hopper listens for standard outgoing cue fire events.

  4. 4

    Run the session

    Press play and work the show as usual. Each time a cue fires on EOS, Hopper seeks the video to the matching moment from your recording so designers and stage management stay aligned with the operator.

Features

Built for accuracy, rehearsal rooms, and real EOS workflows.

Frame-accurate cue sync

Hopper bisects between video frames to locate the precise moment each cue appeared on screen—typically within one frame at 25 fps. Seeks are frame-accurate, not keyframe-approximate.

Automatic cue detection

The built-in OCR engine reads burnt-in cue numbers directly from the video using Apple’s Vision framework. Common display formats are detected automatically—plain numbers, Q-prefixed, “Cue X Label 3.8s” overlays, and more.

Rehearsal recording support

If a cue was rehearsed multiple times in the same recording, every instance is indexed. The cue list shows all occurrences so you can jump to any specific run-through.

Multi-act sequences

Load Act 1, Act 2, and beyond as a single session. Hopper offsets all timestamps globally so cue navigation works seamlessly across files.

Multitrack audio support

Select which audio tracks to play back. Useful for recordings with isolated stems—band, vocals, effects—so the room hears only what’s relevant.

ETC EOS compatible

Works with EOS Family consoles (Eos, Ion, Element, and others) using standard OSC output. UDP on port 8000 or TCP on ports 3032/3037. No console modifications or special session files required.

Sidecar files

Cue data is stored in a small .hopper file alongside your video. Analysis only needs to run once. The sidecar travels with the video and can be stored separately if the video lives on a shared drive.

Native macOS

Built entirely in Swift and SwiftUI. No Electron, no web views, no runtime dependencies. Runs natively on Apple Silicon and Intel Macs running macOS 13 or later.
