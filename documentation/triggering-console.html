Triggering the console

As well as following your ETC EOS console, Hopper can work in the opposite direction — sending OSC cue fire messages to the console as the video plays, keeping the lighting state in sync with the recording. This is currently supported with ETC EOS only.

Console modes

The toggle button in the main Hopper window switches between two console modes:

  • Follow Console. The default. Hopper listens for incoming OSC cue fire messages from the console and jumps the video to the corresponding timestamp. The console operator drives the video.
  • Trigger Console. Hopper drives the console. As the video plays and the playhead crosses each cue's start time, Hopper sends an OSC fire message to EOS. The video operator drives the console.

The two modes are mutually exclusive. Hopper remembers your last active mode and restores it on launch.

Setup

To send triggers to EOS you need to tell Hopper where to reach the console:

  • TCP (recommended). If you have already set up a TCP connection in the Console settings tab, Hopper reuses that same connection for outbound triggers — no extra network configuration is needed. Switch the transport to TCP, enter your EOS IP, and connect.
  • UDP. Open Settings → Remote Control and enter the Console IP and Console port (EOS default is 3032). These are the address Hopper sends outbound UDP packets to, separate from the port Hopper itself listens on for incoming messages.

On the EOS console, ensure OSC RX (receive) is enabled so it will accept incoming fire commands from Hopper.

Cue fire messages

The Start cue trigger field in Settings → Remote Control defines the OSC address template sent each time a new cue is reached. The token {cue} is replaced with the cue number at runtime.

The ETC EOS default:

/eos/cue/1/{cue}/fire

For example, when the playhead reaches cue 40, Hopper sends: 

/eos/cue/1/40/fire

Hopper only sends a message when the cue number changes — it does not fire repeatedly for the same cue while the video plays through it. If you need to target a different cue list, change the 1 in the template to the appropriate list number.

Cue numbers are matched as strings, so fractional EOS cues such as 40.5 are sent and received exactly as-is.

Stop trigger

The Stop trigger field specifies an OSC address to send when Hopper's playback stops — for example when you press pause or the video reaches its end. This is useful for cueing the console to hold state or stop a running sequence.

The ETC EOS default:

/eos/cues/stop

Leave the field blank to disable the stop trigger entirely.

Re-send cue fire on resume

When you pause and then resume playback, the console may have moved on — for example, an operator stepped through cues manually during the break. Enable Re-send cue fire on resume in Settings → Remote Control to have Hopper immediately fire the last cue at the current playhead position as soon as playback resumes, snapping the console back in sync before the next cue arrives.

This fires exactly once per resume event, using the same cue fire template as normal playback. It does not interfere with the stop trigger and does not cause a double-fire when the video continues past the same cue.

Macro triggers

Hopper can listen to EOS macro fire messages and map them to playback actions, allowing a lighting operator to control Hopper from the console without touching the Mac. Available actions:

  • Play — start playback.
  • Stop — pause playback.
  • Playback toggle — play if paused, pause if playing.
  • Follow on — switch to Follow Console mode.
  • Follow off — switch to Trigger Console mode.

In Settings → Remote Control → Macro Triggers, enter the EOS macro number for each action you want to use. No console-side programming is needed beyond creating a blank macro — Hopper sees the macro fire message and acts on it. Leave a field blank to leave that action unbound.

Macro triggers work in both Follow Console and Trigger Console modes. They are a convenient way to hand transport control to the board op so the video operator can focus on the screen.
