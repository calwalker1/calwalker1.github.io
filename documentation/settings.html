Settings

Open Settings from the Hopper menu (or ⌘,). There are four tabs: General, Console, Remote Control, and Playback.

General

Appearance. Choose between System (follows macOS), Light, or Dark. Hopper always switches to Dark when entering fullscreen regardless of this setting, so the interface does not compete with the video.

Console

These settings control how Hopper listens for incoming OSC messages from your lighting console.

Console type. Select the brand of console you are working with. Currently ETC EOS, grandMA3, and a generic Other mode are supported. The console type changes which OSC address patterns Hopper expects and whether the Trigger Console feature is available.

Transport (ETC EOS only). Choose UDP or TCP. UDP is the simpler option — the console broadcasts to a port and Hopper listens. TCP has Hopper connect outbound to your EOS console's IP address; the connection is more reliable and is ETC's recommended approach for production. EOS TCP port is typically 3032 (OSC 1.0) or 3037 (OSC 1.1); Hopper auto-detects the framing.

EOS IP Address (TCP only). The IP address of your EOS console on the local network. Hopper includes an auto-discovery tool — click Discover to scan and fill this in automatically if your console is on the same subnet.

Port. The UDP or TCP port Hopper listens on (default 8000 for UDP). Must match what your console is configured to send to.

Cue list (ETC EOS UDP only). Filters incoming cue fire messages to a specific EOS cue list number. Leave blank to accept messages from all lists. Most shows use list 1.

Address Pattern (grandMA3 / Other). A regular expression with one capture group that Hopper uses to extract the cue number from incoming OSC addresses. Use the OSC Monitor (⌘⇧M) to observe live messages and work out the pattern for your console.

After changing connection settings, click Restart OSC Listener (UDP) or Test Connection (TCP) to apply.

Remote Control

The Remote Control tab has two sections: Trigger Console and Macro Triggers. For a full walkthrough of using these features see Triggering the console.

Trigger Console. Configures Hopper to send OSC fire messages back to your EOS console as the video plays, keeping the console in sync with the recording. Currently only supported with ETC EOS.

  • Console IP / Console port (UDP only). The destination address for outbound trigger messages. Not needed when using TCP — Hopper reuses the existing inbound connection.
  • Start cue trigger. OSC address template sent when a new cue is reached during playback. {cue} is replaced with the cue number at runtime. ETC EOS default: /eos/cue/1/{cue}/fire.
  • Stop trigger. OSC address sent when playback stops. Leave blank to disable. ETC EOS default: /eos/cues/stop.
  • Re-send cue fire on resume. When enabled, resuming playback immediately fires the last cue at the current playhead position. Useful for re-syncing the console state after a pause or a manual seek.

Macro Triggers. Hopper can listen for EOS macro fire messages and map them to actions (play, stop, playback toggle, follow on/off). Enter the EOS macro number for each action; leave blank to disable. No console-side programming is required beyond creating a blank macro — just fire it from a button or the command line.

Playback

Pre-roll. When the video is already playing and Hopper jumps to a cue (via OSC fire or manual navigation), it can seek to a point slightly before the cue time so the console operator sees the video approaching the cue naturally. Two independent pre-roll values are available:

  • Console (OSC) — applied when a cue jump is triggered by an incoming OSC fire message.
  • Manual (cue list / keyboard) — applied when you navigate via the cue list or keyboard shortcuts.

Pre-roll is only applied while the video is playing. If paused, cue jumps always go to the exact cue time. The maximum pre-roll value is 10 seconds.
