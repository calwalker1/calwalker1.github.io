Creating a project and loading your first video

This walkthrough covers starting work in Hopper, attaching an archival recording, and what happens on disk before you draw a cue region and run analysis.

New project

Launch Hopper from Applications. Use the app’s flow to create or open a project (exact menu labels follow the shipping build). A project is the container for your loaded media, optional multi-video sequence, display name, and analysis data.

If you only need a single file for now, you can still start from one video and add more acts or files later — Hopper supports multi-video sequences for productions that span multiple recordings.

Choosing your video

Use the file picker to choose a rehearsal or performance capture. Prefer formats your Mac plays well in QuickTime (see Supported formats on the About page). Very large files are fine as long as you have enough disk space for any temporary work the analyser performs.

After the file loads, you should see playback in the main window. Pause on a frame where the burnt-in cue display is visible before defining the OCR region.

Save location and sidecar

Hopper keeps your video file where you put it. The first time you save analysis or project metadata, the app writes or updates a .hopper sidecar next to that video (same base name). You can move both files together if you relocate the media.

For shared storage (NAS or Dropbox), keep the video and .hopper file in the same folder so collaborators open the same cue map.

Next steps

  • Draw a rectangle over the on-screen cue number region.
  • Run analysis so Hopper can build a cue-to-time map from OCR.
  • Optionally connect your ETC EOS console OSC output for follow-along playback.

For licensing and what lives inside the sidecar file, see About Hopper.
