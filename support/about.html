About Hopper

Hopper is a native macOS app for rehearsing and reviewing show recordings with cue-accurate navigation and optional sync to an ETC EOS console over OSC.

Licensing

Hopper is available under a yearly license. Full functionality is available without a license; unlicensed use shows periodic reminders in the playback window. A valid license removes those reminders for the term of the license. See the Pricing section on the main site for current pricing and terms (including device limits and activation requirements).

Project structure

A Hopper “project” ties together one or more video files (for example across acts), optional display names and act labels, cue-analysis data, and audio-track preferences. You work with media on disk; Hopper does not duplicate your video inside the app bundle.

Cue maps and settings are stored in sidecar files next to your media so they are easy to back up, share on a drive, or exclude from version control if you prefer.

Hopper file

Analysis results and related metadata are saved in a small JSON sidecar with the .hopper extension, alongside the video (same base name). That file records items such as the video path, fractional cue region (so it survives resolution changes), cue identifiers mapped to timestamps, and other saved preferences.

Conceptually, the sidecar looks like this:

{
  "videoPath": "/path/to/show.mp4",
  "cueRegion": { "x": 0.1, "y": 0.05, "width": 0.2, "height": 0.08 },
  "cues": {
    "1": 12.5,
    "40": 1823.0
  }
}

Cue numbers are stored as strings so fractional EOS-style cues (for example 40.5) are preserved. You typically only need to analyse once per source unless you change the region or source file.

Supported formats

Hopper uses Apple’s AVFoundation for playback. In practice that means common container formats such as MOV and MP4 with widely used video codecs (for example H.264 / AVC, HEVC when supported by your Mac) work reliably. ProRes and other professional codecs are often supported when macOS can decode them.

If a file does not open or plays without audio/video, re-export from your NLE or transcode to a standard MP4/MOV profile and try again. Multichannel or multi-track audio is supported for selection in the app where the file exposes multiple audio tracks.
